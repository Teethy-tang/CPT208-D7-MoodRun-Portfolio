2. User Requirements

Structured by coursework requirements: User Journey Map, Requirements List, Evidence of Life.

User Journey Map (Current Experience)

Before Run: User plans to run but hesitates after study/work pressure.

Start: Unsure about intensity and worries about failing to complete goals.

During Run: Pace drops when mood declines; generic prompts feel irrelevant.

After Run: Sees numbers but lacks emotional reflection and motivation for next run.

Pain Points Summary

  • No simple way to convert current mood into an achievable running plan.
  • Most feedback systems prioritize performance over emotional readiness.
  • Post-run data does not help users understand motivation fluctuations.

Requirements List

Three must-haves for the system to be considered playful.

Must-Have Playful Requirements

  • R1: 5-second mood check-in that immediately changes the recommended run mode.
  • R2: In-run adaptive prompts (tone, frequency, challenge) based on user state.
  • R3: Post-run playful reflection card with encouragement and next-step suggestion.

Priority Rationale

  • R1 reduces decision friction before running starts.
  • R2 keeps support relevant during emotional changes.
  • R3 reinforces habit continuity instead of single-session performance.
